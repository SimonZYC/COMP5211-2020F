COMP5211 Project

Capture the Flag Competition

This project is modified from the Berkeley AI Pacman game. In this project, you will implement a Pacman agent and compete with other classmates on the capture the flag game. In the game, your Pacman agent needs to eat a food dot assigned and avoids collision with other agents.

You can find the skeleton code in project.zip.

Game Rules

Layout

Two agents are placed randomly on an 8 by 8 gameboard. Each agent needs to navigate to a randomly generated food dot. The initial (Manhattan) distance between the pair of agents is at least 3 cells. The initial distance between the agent and its assigned food dot is at least 5 cells.

Scoring

An agent can earn 1 point if it reaches its goal with the minimum number of steps mmm﻿, i.e. the agent walks on the shortest path. Otherwise, it earns ms\frac{m}{s}sm​﻿ points if it reaches the goal with sss﻿ steps. If the agent fails to reach the goal within the time limit, it gets 0 scores. Besides, if the agent collides with the other agent, both agents lose immediately and get -1 scores. Note that a collision can happen even if the agent has reached the goal. The same penalty will be applied. Demos.

Observations

In each round, the agent receives 5 pieces of information:

position: the current position of agent target: a list of the position of the target food dot (either a list of one item or an empty list) sensor: a list of 8 sensor values [s0,s1,...,s7][s_0,s_1,...,s_7][s0​,s1​,...,s7​]﻿ for wall detection (sj=1s_j=1sj​=1﻿ if there is a wall in that direction) radar: a list of the position of other agents within 3 cells score: the score obtained so far

Winning

In a match (your agent vs another agent), we will play 10 games. The team with the higher total score over the 10 games will win. Your agent will compete in multiple matches. In each match, you get 3 points for a win, 1 point for a tie, 0 point for a loss. The final rank in a tournament depends on the total points.

Constraints

Computation Time The games will be run on the TA's laptop with an i7-6500U CPU (no GPUs provided). You are required to finish each of your moves within 1 second. However, there will be an initial start-up allowance of 15 seconds (use the registerInitialState function). Agents that compute during the opponent's turn will be disqualified. In particular, any form of multi-threading is disallowed.

Environments Different from the previous assignments, which we encourage implementing from scratch with Python 2.7. This project will be run on the Python 3.7 environment. The skeleton code itself does not require additional packages. However, you are allowed to use the following packages:

numpy==1.18.5

tensorflow==2.3.1

Since we will run your agent with other classmates' agents in the same environment, please tie to the environment provided. If you find other packages useful, you may suggest it to the TA. We will try to accommodate your requests as long as it does not conflict with the mentioned environment.

Getting started

By default, you can run a game with the simple baselineTeam that we have provided:

python capture.py

A wealth of options are available to you:

python capture.py --help

There are multiple slots for agents. Agents 0 is on the red team, and agent 1 is on the blue team. We implemented an example in the baselineTeam. It is chosen by default as both the red and blue teams. You can obtain the default setting by:

python capture.py --agents baselineTeam baselineTeam

which specifies that the agents 0 and 1 are both created from baselineTeam.py.

To control one of the two agents with the keyboard, pass the appropriate option:

python capture.py --keys0

python capture.py --keys0 --keys1

You can control the first agent with w, a, s, d, and the second agent with i, j, k, l.

Layout

By default, all games are run on the defaultcapture layout. To test your agent on other layouts, use the -l option. In particular, you can generate random layouts by specifying RANDOM[seed]. For example, -l RANDOM13 will use a map randomly generated with seed 13.

Recordings

You can record local games using the --record option, which will write the game history to a file named by the time the game was played. You can replay these histories using the --replay option and specifying the file to replay.

Designing Agent

Baseline Team

To kickstart your agent design, we have provided you with a baseline agent, defined in baselineTeam.py. The agent is quite naive. It goes straight in one direction, turns once, then goes straight to the goal. There is quite a chance of collision if both agents employ the same strategies. Please take the baseline agent as a reference and design a better agent!

File Format

You should include your agent in a file of the same format as in myTeam.py. You should submit your work as [studentID].py (e.g. 12345678.py). Your agents must be completely contained in this one file.

Interface

In myTeam.py, we provide a minimum implementation. Your agent should be a subclass of the CaptureAgent. You can use the registerInitialState method to initialize your agent and chooseAction method to decide which action your agent would take. Both methods take on an argument state which provides information mentioned previously.
